# Финальное ТЗ v1.4

## Executive Summary

Разрабатывается Telegram-бот **EnergyCFO | Встречи** для согласования B2B-встреч с Татьяной Коваль. Пользователь попадает в бота с сайта или по прямой ссылке, выбирает тему встречи, оставляет данные, выбирает свободный слот и отправляет заявку. Встреча создается в Google Calendar только после ручного согласования администратором.

MVP реализуется как Telegram-чат-бот с админ-функциями внутри Telegram. Mini App, веб-админка, Google Sheets-отчетность, CRM и оплата выносятся в следующие версии. Размещение: Beget Cloud VPS `energycfo`, домен `energycfo.pro`, поддомен `meet.energycfo.pro`, Google-аккаунт `koval.tatyana.2502@gmail.com`.

## 4. Пользовательские сценарии

### 4.1. Создание заявки

1. Пользователь открывает бота с сайта или по прямой ссылке.
2. Бот показывает приветствие и запрашивает согласие на обработку данных.
3. Пользователь выбирает тему встречи:
   - Корпоративные функции;
   - Тарифная кампания;
   - Комплексный подход;
   - Другое.
4. При выборе «Другое» пользователь вводит тему вручную.
5. Пользователь вводит ФИО, компанию, должность, email, комментарий.
6. Пользователь выбирает предпочтительный способ связи: Telegram, email, телефон, WhatsApp, другое.
7. Телефон запрашивается только если выбран телефон или WhatsApp.
8. Бот показывает доступные даты и слоты на 2 месяца вперед.
9. Пользователь выбирает слот.
10. Бот показывает итог заявки.
11. Пользователь подтверждает отправку.
12. Заявка получает статус «Ожидает согласования».
13. Администратор получает уведомление.

### 4.2. Отмена встречи пользователем

1. Пользователь открывает активную встречу.
2. Нажимает «Отменить встречу».
3. Система проверяет, что до встречи больше 12 часов.
4. Если условие выполнено, встреча отменяется автоматически.
5. Событие в Google Calendar отменяется.
6. Администратор получает уведомление.

### 4.3. Перенос встречи пользователем

1. Пользователь нажимает «Перенести встречу».
2. Система проверяет, что до встречи больше 12 часов.
3. Бот показывает ближайшие свободные слоты.
4. Пользователь выбирает новый слот.
5. Создается заявка на перенос.
6. Старая встреча остается действующей до решения администратора.
7. После согласования событие в Google Calendar обновляется.
8. При отказе старая встреча остается без изменений.

## 5. Админ-сценарии

### 5.1. Обработка заявки

1. Администратор получает карточку заявки.
2. В карточке отображаются: ФИО, компания, должность, email, Telegram-профиль, предпочтительный канал связи, тема, комментарий, слот.
3. Администратор выбирает действие: согласовать, отклонить, предложить другое время.
4. При согласовании создается событие в Google Calendar.
5. При отказе причина обязательна.
6. При предложении другого времени администратор может отправить до 3 альтернативных слотов.

### 5.2. Управление доступностью

1. Администратор открывает раздел доступности.
2. Бот показывает календарь на 3 месяца вперед.
3. Администратор отмечает даты, когда встречи не принимаются.
4. Исключенные даты полностью скрываются от пользователя.
5. Администратор может менять рабочие часы, длительность, буфер, лаг и лимит встреч.

### 5.3. Напоминания администратору

| Событие | Действие |
|---|---|
| Заявка не обработана 2 часа | первое напоминание |
| Заявка не обработана 12 часов | второе напоминание |
| Заявка не обработана 24 часа | третье напоминание, статус «Просрочена по SLA» |
| До выбранного времени осталось менее 12 часов | несогласованная заявка автоотменяется |
