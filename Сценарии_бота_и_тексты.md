# Сценарии Telegram-бота и тексты сообщений

Документ содержит пользовательские и админские сценарии Telegram-бота **EnergyCFO | Встречи**, тексты сообщений, кнопки и статусы. Тексты являются рабочей версией для MVP и могут быть уточнены после тестирования.

## 1. Общие принципы текстов

| Принцип | Как применяем |
|---|---|
| Кратко и по делу | без длинных объяснений и канцелярита |
| Деловой тон | B2B-аудитория, финансовые и энергетические задачи |
| Прозрачный статус | пользователь всегда понимает, что происходит с заявкой |
| Без технических деталей | ошибки API и сервера не показываются пользователю |
| МСК везде | время отображается с указанием МСК |
| Без давления | пользователь сам выбирает удобный слот и канал связи |

## 2. Основные кнопки пользователя

| Кнопка | Назначение |
|---|---|
| `Назначить встречу` | старт создания заявки |
| `Мои встречи` | список активных заявок и встреч |
| `Помощь` | краткая справка |
| `Отменить` | отмена текущего действия |
| `Назад` | возврат на предыдущий шаг |
| `Подтвердить заявку` | отправка заявки администратору |
| `Изменить данные` | возврат к редактированию |
| `Отменить встречу` | отмена согласованной встречи |
| `Перенести встречу` | запрос переноса |

## 3. Основные кнопки администратора

| Кнопка | Назначение |
|---|---|
| `Новые заявки` | список заявок на согласование |
| `Заявки на перенос` | переносы, ожидающие решения |
| `Активные встречи` | согласованные будущие встречи |
| `Недоступные даты` | исключение дат из записи |

| `Отклонить` | отказать с причиной |

## 4. Команда `/start`

### Условие

Пользователь впервые открывает бота или возвращается к главному меню.

### Текст

```text
Здравствуйте. Это бот для записи на встречу с Татьяной Коваль.

Здесь можно выбрать тему встречи, свободное время и отправить заявку на согласование. После подтверждения встреча появится в Google Calendar, а ссылка Google Meet придет в Telegram.

Выберите действие:
```

### Кнопки

| Кнопка |
|---|
| `Назначить встречу` |
| `Мои встречи` |
| `Помощь` |

| Кнопка |
|---|
| `Админ-панель` |

## 5. Согласие на обработку данных

Юридический текст отложен, но в MVP должен быть технический шаг согласия перед сбором данных.

### Текст-заглушка до юридического согласования

```text
Перед записью нужно подтвердить согласие на обработку данных, которые потребуются для организации встречи: ФИО, компания, должность, email, тема встречи, комментарий и выбранный способ связи.

Точный текст согласия и ссылка на политику обработки персональных данных будут добавлены перед промышленным запуском.
```

### Кнопки

| Кнопка | Действие |
|---|---|
| `Согласен и продолжить` | переход к выбору темы |
| `Не согласен` | завершение сценария |

### Текст при отказе

```text
Без согласия бот не может принять заявку на встречу.

Если хотите связаться напрямую, используйте контакт, по которому вы получили ссылку на бота.
```

## 6. Выбор темы встречи

### Текст

```text
Выберите тему встречи.
```

### Кнопки

| Кнопка | Значение |
|---|---|
| `Корпоративные функции` | тема по разделу сайта |
| `Тарифная кампания` | тема по разделу сайта |
| `Комплексный подход` | тема по разделу сайта |
| `Другое` | пользователь вводит тему вручную |
| `Назад` | возврат |

### Если выбрано `Другое`

```text
Кратко опишите тему встречи.
```

### Валидация

| Проверка | Сообщение |
|---|---|
| Пустой ввод | `Пожалуйста, укажите тему встречи.` |
| Слишком длинный текст | `Тема слишком длинная. Сформулируйте короче, до 300 символов.` |

## 7. Сбор данных пользователя

### 7.1. ФИО

```text
Укажите ФИО.
```

Ошибка:

```text
ФИО обязательно для заявки. Пожалуйста, укажите ФИО.
```

### 7.2. Компания

```text
Укажите компанию.
```

Ошибка:

```text
Компания обязательна для заявки. Пожалуйста, укажите название компании.
```

### 7.3. Должность

```text
Укажите вашу должность.
```

Ошибка:

```text
Должность обязательна для заявки. Пожалуйста, укажите должность.
```

### 7.4. Email

```text
Укажите email для календарного приглашения.
```

Ошибка:

```text
Похоже, email указан некорректно. Проверьте адрес и отправьте еще раз.
```

### 7.5. Комментарий

```text
Кратко опишите вопрос или задачу, которую хотите обсудить.
```

Ошибка:

```text
Комментарий нужен, чтобы оценить контекст встречи. Напишите, пожалуйста, 1-2 предложения.
```

## 9. Выбор даты

### Текст

```text
Выберите дату встречи.

Показываются доступные даты на ближайшие 2 месяца. Время указано по МСК.
```

### Если доступных дат нет

```text
Сейчас нет доступных дат для записи.

Попробуйте позже или свяжитесь напрямую по каналу, где вы получили ссылку на бота.
```

### Правила отображения

| Правило | Поведение |
|---|---|
| Даты только на 2 месяца вперед | дальше не показывать |
| Исключенные даты | не показывать |
| Выходные | не показывать |
| Дни с лимитом 5 встреч | не показывать |
| Дни без свободных слотов | не показывать |

## 13. Согласование заявки

### Действие администратора

Администратор нажимает `Согласовать`.

### Текст администратору при успехе

```text
Заявка #{request_id} согласована.

Событие создано в Google Calendar, пользователю отправлено уведомление.
```

### Текст пользователю

```text
Ваша встреча согласована.

Дата и время: {date_time_msk}
Формат: Google Meet
Ссылка: {google_meet_link}

Календарное приглашение отправлено на email: {email}
```

### Если событие не создалось

Администратору:

```text
Не удалось создать событие в Google Calendar по заявке #{request_id}.

Заявка не потеряна. Проверьте интеграцию с календарем или повторите действие позже.
```

Пользователю техническую ошибку не отправлять, пока администратор не примет решение.

## 14. Отказ по заявке

### Действие администратора

Администратор нажимает `Отклонить`.

### Текст запроса причины

```text
Укажите причину отказа.

В бизнес-среде отказ должен быть понятным для пользователя.
```

### Типовые причины

| Кнопка |
|---|
| `Не подходит тема обращения` |
| `Недостаточно информации` |
| `Нужен другой формат взаимодействия` |
| `Нет доступности в выбранный период` |
| `Другое` |

### Если выбрано `Другое`

```text
Напишите причину отказа.
```

### Текст пользователю

```text
Заявка на встречу отклонена.

Причина: {decline_reason}

При необходимости вы можете создать новую заявку с другой темой или временем.
```

### Текст администратору

```text
Заявка #{request_id} отклонена.

Причина отправлена пользователю.
```

## 18. Перенос встречи пользователем

### Пользователь нажимает `Перенести встречу`

Если до встречи меньше 12 часов:

```text
До встречи осталось менее 12 часов. Перенос через бота недоступен.

Пожалуйста, свяжитесь напрямую по каналу, где согласовывалась встреча.
```

Если до встречи больше 12 часов:

```text
Выберите новый слот для переноса встречи.

Перенос вступит в силу только после повторного согласования.
```

### После выбора нового слота

```text
Заявка на перенос отправлена.

Текущая встреча остается в силе до решения администратора.
```

### Уведомление администратору

```text
Новая заявка на перенос встречи.

Заявка: #{request_id}
ФИО: {full_name}
Компания: {company}
Текущее время: {old_date_time_msk}
Новое время: {new_date_time_msk}

Старая встреча остается действующей до вашего решения.
```

### Кнопки администратора

| Кнопка |
|---|
| `Согласовать перенос` |
| `Отклонить перенос` |

### Если перенос согласован

Пользователю:

```text
Перенос встречи согласован.

Новое время: {new_date_time_msk}
Ссылка Google Meet: {google_meet_link}

Календарное событие обновлено.
```

Администратору:

```text
Перенос по заявке #{request_id} согласован.

Событие в Google Calendar обновлено.
```

### Если перенос отклонен

Пользователю:

```text
Перенос встречи отклонен.

Текущая встреча остается в силе:
{old_date_time_msk}

Причина: {decline_reason}
```

Администратору:

```text
Перенос по заявке #{request_id} отклонен.

Старая встреча остается действующей.
```

Вы можете выбрать другой доступный слот.
```

3. После подтверждения вы получаете ссылку Google Meet и календарное приглашение.

Время указано по МСК.
```

## 26. Технические ошибки

### Пользовательский текст

```text
Не удалось выполнить действие.

Попробуйте еще раз позже. Если ошибка повторится, свяжитесь напрямую по каналу, где вы получили ссылку на бота.
```

### Админский текст

```text
Техническая ошибка.

Раздел: {module}
Действие: {action}
Заявка: {request_id}
Время: {timestamp}

Проверьте журнал ошибок.
```

## 27. Статусы и пользовательские формулировки

| Технический статус | Текст для пользователя |
|---|---|
| `draft` | Черновик |
| `pending_approval` | Ожидает согласования |
| `approved` | Согласована |
| `declined` | Отклонена |
| `cancelled_by_user` | Отменена |
| `reschedule_pending` | Ожидает согласования переноса |
| `rescheduled` | Перенесена |
| `sla_overdue` | Требует решения |
| `completed` | Завершена |

## 29. Критерии приемки текстов

| Проверка | Критерий |
|---|---|
| Все ключевые сценарии имеют текст | да |
| У каждого действия есть понятная кнопка | да |
| Пользователь не видит технические ошибки | да |
| Время указано в МСК | да |
| Отказ содержит причину | да |
| Перенос явно требует согласования | да |
| Отмена меньше чем за 12 часов запрещена | да |
| Согласование сообщает про Google Meet | да |
