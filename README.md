# EnergyCFO | Встречи

Стартовый комплект документации для разработки Telegram-бота согласования B2B-встреч с Татьяной Коваль.

Бот нужен для автоматизации записи на деловые встречи: пользователь переходит с сайта или по прямой ссылке в Telegram, выбирает тему, оставляет данные, выбирает свободный слот, после чего заявка уходит администратору на согласование. Встреча создается в Google Calendar только после ручного подтверждения.

## 1. Что входит в проект

| Блок | Описание |
|---|---|
| Telegram-бот | пользовательский сценарий записи и админ-согласование |
| Google Calendar | проверка занятости, создание, перенос и отмена событий |
| Google Meet | автоматическая ссылка на онлайн-встречу |
| Админка в Telegram | заявки, переносы, исключенные даты, настройки |
| PostgreSQL | хранение пользователей, заявок, встреч и настроек |
| Фоновые задачи | SLA, напоминания, автоотмена несогласованных заявок |
| Beget VPS | размещение backend и базы данных |

## 5. Порядок старта разработки

1. Создать Telegram-бота через BotFather.
2. Зафиксировать username бота.
3. Получить Telegram bot token и сохранить его как секрет.
4. Проверить SSH-доступ к серверу Beget.
5. Настроить `Nginx` и SSL для `meet.energycfo.pro`.
6. Создать репозиторий проекта.
7. Подготовить `.env.example` без секретов.
8. Поднять базовый backend.
9. Подключить Telegram webhook.
10. Подключить PostgreSQL.
11. Реализовать пользовательский сценарий заявки.
12. Реализовать админ-согласование.
13. Подключить Google Calendar и Google Meet.
14. Добавить расчет слотов и бизнес-ограничения.
15. Добавить напоминания и фоновые задачи.
16. Провести приемочное тестирование.
17. Запустить MVP.

## 6. Локальный запуск backend

```powershell
npm.cmd install
npm.cmd run typecheck
npm.cmd test
npm.cmd run build
npm.cmd run dev
```

Проверка работоспособности:

```powershell
Invoke-RestMethod http://127.0.0.1:3000/health
```

Файл `.env.example` содержит только безопасные заглушки. Реальный `.env` создается локально или на сервере и не коммитится. Telegram token, Google client secret, Google refresh token и пароль БД в Markdown не записываются.

## 7. Что не публиковать

| Данные | Причина |
|---|---|
| Telegram bot token | дает управление ботом |
| Google OAuth secret | дает доступ к интеграции |
| Refresh token Google | дает доступ к календарю |
| Пароли | не должны передаваться и храниться в документах |
| `.env` | содержит секреты production |
| Бэкапы БД | могут содержать персональные данные |

## 9. Definition of Done для MVP

MVP считается готовым, если:

| Блок | Критерий |
|---|---|
| Пользователь | может создать заявку от начала до конца |
| Администратор | может согласовать, отклонить, предложить другое время |
| Google Calendar | событие создается, переносится и отменяется |
| Google Meet | ссылка создается автоматически |
| Слоты | учитывают лаг, буфер, лимит, занятость и исключенные даты |
| Напоминания | уходят пользователю и администратору |
| Ошибки | логируются и отправляются администратору |
| Бэкапы | настроены и проверены |
| Production | бот работает через `https://meet.energycfo.pro` |
