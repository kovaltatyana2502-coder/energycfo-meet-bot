## 4. Telegram BotFather

### Что нужно сделать

| Шаг | Действие | Результат |
|---|---|---|
| 1 | Открыть `@BotFather` в Telegram | открыт официальный бот |
| 2 | Отправить `/newbot` | начато создание бота |
| 3 | Указать имя | `EnergyCFO | Встречи` |
| 4 | Указать username | например `energycfo_meet_bot` |
| 5 | Получить token | сохранить как секрет |
| 6 | Проверить ссылку | бот открывается в Telegram |

### Варианты username

| Вариант | Оценка |
|---|---|
| `energycfo_meet_bot` | основной рекомендуемый |
| `energycfo_calendar_bot` | понятный, но длиннее |
| `koval_meet_bot` | персональный вариант |
| `cfo_energy_meet_bot` | альтернативный вариант |

Username должен заканчиваться на `bot`.

### Что зафиксировать после создания

| Данные | Где хранить |
|---|---|
| Username бота | можно в документации |
| Ссылка на бота | можно в документации |
| Telegram bot token | только в секретах, не в Markdown |
| Telegram ID администратора | в `.env` |

### Что нужно сделать в Google Calendar

| Шаг | Действие |
|---|---|
| 1 | Войти в Google Calendar под `koval.tatyana.2502@gmail.com` |
| 2 | Создать отдельный календарь `Встречи с сайта CFO Energy Advisory` |
| 3 | Проверить, что календарь отображается в списке |

| Запрещено | Почему |

| Публиковать OAuth secret | риск доступа к интеграции |
| Хранить refresh token в документации | риск доступа к календарю |

## 7. Переменные окружения

Перед запуском должны быть подготовлены значения:

| Переменная | Статус |
|---|---|
| `NODE_ENV` | определить |
| `APP_BASE_URL` | готово: `https://meet.energycfo.pro` |
| `TELEGRAM_BOT_TOKEN` | после BotFather |
| `TELEGRAM_ADMIN_ID` | получить |
| `DATABASE_URL` | после настройки PostgreSQL |
| `GOOGLE_CLIENT_ID` | после Google Cloud |
| `GOOGLE_CLIENT_SECRET` | после Google Cloud |
| `GOOGLE_REDIRECT_URI` | после настройки backend |
| `GOOGLE_CALENDAR_ID` | после создания календаря |
| `TIMEZONE` | `Europe/Moscow` |

Подробности: [Переменные_окружения_env.md](Переменные_окружения_env.md).

## 12. Что сделать первым

Рекомендуемый ближайший порядок:

1. Создать Telegram-бота в BotFather.
2. Записать username бота.
3. Не публикуя token, подготовить его для `.env`.
4. Получить свой Telegram ID.
5. Создать репозиторий проекта.
6. Перенести документы в `docs/`.
7. Создать `.env.example`.
8. Начать разработку каркаса backend.
